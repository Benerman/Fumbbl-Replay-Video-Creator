FUMBBL Replay Video Creator

A toolchain for turning a Blood Bowl replay on FUMBBL into a short whimsical highlight reel - this page shows where the project is today.

What it does

Give it a FUMBBL match id (or a URL or a JNLP file). It pulls down the match summary and the full per-turn event log, finds the plays that mattered most, and prints a ranked report. With --tableaux DIR it also renders a PNG per pivotal play; with --gifs DIR it renders an animated GIF of the run-up.

How the pipeline works

1

Resolve replay reference

jnlp_loader.resolve() turns a URL, .jnlp file, or bare numeric id into a match id.

2

Fetch summary + replay

/api/match/get/{id} gives the scoreboard; /api/replay/get/{replayId}/gz streams the full gzipped game log over plain HTTP - the same per-turn deltas the official Java client gets over its websocket.

3

Walk the gameLog

events.extract_events() walks gameLog.commandArray, holds onto sticky state (half, per-team turn, running score), and emits one typed Event per scoring or casualty counter increment - each stamped with the post-event score, the player ids, and the inflicter for casualties.

4

Score pivotal plays

analyzer.analyze() applies context modifiers - a TD that pushes the eventual winner past the loser's final score is "game-winning", a TD that levels the score is "tying", a kill from a foul is rated higher than a routine block. Headlines name the player and turn.

5

Reconstruct the field

For visuals we rebuild the on-pitch state at any moment: field_state.reconstruct_at(replay, cmd_nr) replays every fieldModelSetPlayerCoordinate / fieldModelRemovePlayer / fieldModelSetBallCoordinate delta up to that command. A stop_at hook halts mid-command - needed because the post-TD cleanup that sweeps every player to the dugout fires in the same command as the score itself.

6

Render

tableau.render_tableau() draws a single still PNG for one play; animate.render_play_gif() stitches frames from the run-up into an animated GIF. Both highlight the involved players (scorer, victim, inflicter) with a yellow ring.

The image format

The pitch is the FFB 26x15 grid. Endzones are the leftmost and rightmost columns; the line of scrimmage is the bright line dead-centre. Tokens use team colour and the player's jersey number; the ball is a small white circle wherever it was last placed.

Try it

pip install -r requirements.txt

# Text report for any match id
python -m fumbbl_replay 4700842

# JSON for downstream tooling
python -m fumbbl_replay 4700842 --json

# Render PNG tableaux
python -m fumbbl_replay 4700842 --tableaux out/tableaux

# Render animated GIF per pivotal play
python -m fumbbl_replay 4700842 --gifs out/gifs
